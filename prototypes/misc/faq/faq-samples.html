Do you have a plan to move to Bootstrap 4?

In parts, Bootstrap 4 is already available. You can read more about what we've added so far at the UX Library.

The reasons why we're keeping much of the Bootstrap 3 classes, like for example panels, is that we don't want to force dozens of applications to re-factor their code just to get their hands on some nice Bootstrap 4 features.

In the upcoming Corporate UI 4.0, we've pretty much decoupled the CSS so it will be easier to make your own choice of what framework to use.

What's the latest version of Corporate UI, and where can I find any release notes?

The best way to find out more about the Corporate UI releases is to look in the release section of the Corporate UI GitHub repository. The latest release is also the latest version.

How do I know what Corporate UI version a web page currently is using?

When Corporate UI is correctly installed, the version number will be printed on the top of the code in the code inspector. To open the code inspector, press F12 on your keyboard. The corporate-ui-version should now be visible on the second line of code. If you don't see it, press the Elements tab in the new window. Alternatively, the web page does not use Corporate UI.

What coding conventions are you using for Corporate UI?

We think that it's important to share common standards within a company. Therefore, we've done some research and the standards we're using when coding is published here:

https://github.com/scania/coding-conventions/

What browsers are supported?

The Corporate UI supports most modern browsers. When it comes to Internet Explorer and Microsoft Edge, all versions above IE10 is supported. When it comes to Chrome, version 49 is required and when it comes to FireFox well test in version 40 and above. For Safari on Mac, version 8 or better is required.

You'll find out the details of the complete rules in the browser-reject.js file.

To reject old browers, you'll need to include this script in your head section:

<script src="https://static.scania.com/vendors/components/pure-js/browser-reject/1.0.0/js/browser-reject.js"></script>

What frameworks and libraries are used for Corporate UI?

There are several generations of Corporate UI currently in production and they use different kind of techniques to serve you. The early versions used jQuery and Bootstrap only, generation 2 is using jQuery, Polymer 1.4 togheter with Scania Bootstrap. In generation 3, the jQuery dependency was removed and the JavaScript part was replaced by Bootstrap Native.

In Corporate UI 4, we're removing all dependencies to other libraries, delivering native JavaScript code only. This decreases the risk for conflicts with the frameworks and libraries that you're using to develop your application. The tool that we're using to create the native JavaScript is StencilJS. More information about StencilJS is available here: https://stenciljs.com/docs/introduction.

Corporate UI Components

Corporate Header

Example

Optional User Menu 

<c-corporate-header site-name="My application name" short-name="My app" site-url="/">
  <nav-item location="#">Optional</nav-item>
  <nav-item location="#">User Menu</nav-item>
</c-corporate-header>

Site name and Short name

Add a site-name and a short-name that will be displayed in mobile mode. The site name will be displayed on the right hand side of the logotype on desktop mode and top centered in mobile mode.

Header links

You can add one or several nav-item to the Corporate Header. Typically these are used to link away from the site, to for example Scania.Com. This location is also used for language selectors and user menus.

Dealer variation

Dealers have a slightly different header setup as they need to include their name and/or their logotype.

Configurator 

<c-corporate-header dealer-name="Bilmetro" short-name="Bilmetro"  dealer-url="/" dealer-logo="https://scania.github.io/corporate-ui-docs/component/examples/bilmetro-logo.png">
  <nav-item location="#">Configurator</nav-item>
</c-corporate-header>

Main Content

The sole purpose of the <c-main-content> element is to calculate the distance between the Corporate Header and the Corporate Footer. Using <c-main-content> on your page will ensure you that the footer never slides up the page. If the acutal content on a page is not tall enough to push the footer down and out of the browser viewport, it will be "hanging" on the page with whitespace underneath. 


<c-main-content> 
</c-main-navigation><c-main-navigation>
  <section>
    <div class=" container-fluid">

    Your application goes here. If you are using .container-fluid,

    your page will take the whole browser width. 

    Using .container, it will give you margins on wider screens.

    </div>
  </section>
</c-main-content>

Sections and Containers

While c-main-content is calculating the distance between header and footer, you'll need additional elements to set margins and padding. The <section> element is setting both the background colour as well as top and bottom margins. You can override the standard background colour by using the classes .light or .dark. To set boxed left and right margins, use the .container class. For a fluid design use .container-fluid class. A page can be built using several sections with a variety of containers.

Main Navigation

The main navigation is the element that's located on the very top of this browser window (we can't show another example as you naturally only can have one main navigation a page). It has a desktop mode and a mobile mode where the break point for desktop/mobile is at 992px. If you make your browser window narrower or wider you'll see how the navigation changes its appearance. Below you'll find the code needed for incorporating the navigation.


<c-main-navigation>
  <primary-items>
    <nav-item location="#">Home</nav-item>
    <nav-item location="#">Style Guide</nav-item>
      <sub-navigation caption="Style Guide">
        <nav-item location="#">Logotype</nav-item>
        <nav-item location="#">Icons</nav-item>
      </sub-navigation>
  </primary-items>
  <secondary-items>
     Secondary items are optional, more information below.
  </secondary-items>
</c-main-navigation>

Nav items

Nav items is setting the design of links within our components. You can also use them with properties and attributes. 


<nav-item attrs="{ target: '_blank'}" location="#" props="{ spellcheck: false, onclick: function() { window.open('https://www.facebook.com/scaniagroup/') }" icon=" facebook">

The "More" transformation

Naturally, the horizontal space available for links on the main navigation is limitid. Although, in the case that the do not fit all on one row, the ones that do not fit, will be moved to a drop down meny called "More".

Secondary items (optional)

In the <secondary-items> element you can place any custom markup that your solution demands. A common example is the user profile drop down exampled below. In desktop mode, on this page's main navigation, it's the blue box in the upper right hand side of the browser window. If you're in mobile mode, the user profile menu is avaiable among the navigation items on the mobile menu. 



<secondary-items>
  <div class="dropdown"> 
    <div role="button">                 
      <div class="user-profile user-menu dropdown-toggle" data-toggle="dropdown">
        <div class="user-name hidden-md">   
          <span>Jesper Karlsson<span>
          <small>Company name<small>
        </div>
        <div class="user-profile-picture" style="background-image: url('https://scania.github.io/corporate-ui-docs/images/sample-user-image.jpg')"> 
      </div>  
    </div>
    <ul class="dropdown-menu"aria-labelledby="myDropdown">
      <li role="presentation"><a role="menuitem" tabindex="-1"> href="https://idm.scania.com"> Change Password</a></li>
      <li role="presentation"><a role="menuitem" tabindex="-1"> href="#"> Logout</a></li>
    </ul>        
  </div>
</secondary-items>

Corporate Footer

Basic Example

By default, the <c-corporate-footer> element renders with a copyright text and a wordmark. All you have to do is to include the element as shown in the example.

Corporate Footer with Links and Social Icons

By including the social-icons component in the footer you'll be able to promote your local social channels. Add nav-items if you're in need of footer links.

Opening a link in a new browser window or tab

If you prefere to open a nav-item in a new window, please use the JS event onclick="window.open('https://wherever.com')" to do the trick. It's included in the above code sample.

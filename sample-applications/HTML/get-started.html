Introduction

Get started with Corporate UI, the design system based on the popular Bootstrap framework and that's extended with tailor made Corporate UI Custom Elements.

Corporate UI is 100% open source, available at GitHub.
Via Slack you'll reach the Corporate UI core team as well as many developers using and contributing to the framework.

Quick start

Looking to quickly add Corporate UI to your project? Use our CloudFront CDN. Want to use a package manager? Contact us to participate in the final tests of the NPM package.

Quick Install

Copy-paste the Corporate UI initializer <script> just before the </head> tag.


<script src="https://static.scania.com/build/global/3.5.2/js/corporate-ui.js">

About our JavaScripts

All of our components and custom elements are using native JavaScripts to function. Therefore, there is no need to load extra libraries for it to work. We've replaced Bootstraps jQuery Components with the Bootstrap Native JS Library. Some features is triggered a little bit different but it's an easy upgrade.

State management with Redux

You'll be able to access any data that our custom elements collects or updates via the Corporate UI Redux Store. If you're in need of a more diverse state management overall, please contact us and we can find can find a process that works for you. Learn more...

Important globals

Corporate UI employs a handful of important global styles and settings that you’ll need to be aware of when using it, all of which are almost exclusively geared towards the normalization of cross browser styles. Let’s dive in.

HTML5 doctype

Corporate UI requires the use of the HTML5 doctype. Without it, you’ll see some funky incomplete styling, but including it shouldn’t cause any considerable hiccups.


<!DOCTYPE html>
<html lang="en">
  ...
</html>

Responsive meta tag

Corporate UI is developed mobile first, a strategy in which we optimize code for mobile devices first and then scale up components as necessary using CSS media queries. To ensure proper rendering and touch zooming for all devices, add the responsive viewport meta tag to your <head>.


<meta name="viewport" content="width=device-width, initial-scale=1, shrink-to-fit=no">

Box-sizing

For more straightforward sizing in CSS, we switch the global box-sizing value from content-box to border-box. This ensures padding does not affect the final computed width of an element, but it can cause problems with some third party software like Google Maps and Google Custom Search Engine.

On the rare occasion you need to override it, use something like the following:


.selector-for-some-widget {
  box-sizing: content-box;
}

With the above snippet, nested elements—including generated content via ::before and ::after—will all inherit the specified box-sizing for that .selector-for-some-widget.

Learn more about box model and sizing at CSS Tricks.

Important Javascript files

In the head section we handle what happends both when a user has disabled his browsers Javascript encoding as well as re-directing older browsers to an explaination and instructions of how to install a modern browser.

Custom Elements

The most common interaction patterns at Scania are made into custom elements, to offer easy implementation of sometimes advanced features. They also make it possible to create a coherent user experience over different applications and platforms. Custom elements are similar to framework specific components, except that they're supported by the browsers and normally have no dependencies to other technologies. A custom element works like any other HTML element why they are so easy to adopt.

Corporate Header

Example


<c-corporate-header site-name="My application name" short-name="My app" site-url="/">
  <nav-item location="#">Optional</nav-item>
  <nav-item location="#">User Menu</nav-item>
</c-corporate-header>

Site name and Short name

Add a site-name and a short-name that will be displayed in mobile mode. The site name will be displayed on the right hand side of the logotype on desktop mode and top centered in mobile mode.

Header links

You can add one or several nav-item to the Corporate Header. Typically these are used to link away from the site, to for example Scania.Com. This location is also used for language selectors and user menus.

Dealer variation

Dealers have a slightly different header setup as they need to include their name and/or their logotype.


<c-corporate-header dealer-name="Bilmetro" short-name="Bilmetro"  dealer-url="/">
  <nav-item location="#">Configurator</nav-item>
</c-corporate-header>
                

<c-corporate-header dealer-name="Bilmetro" short-name="Bilmetro"  dealer-url="/" dealer-logo="https://scania.github.io/corporate-ui-docs/component/examples/bilmetro-logo.png">
  <nav-item location="#">Configurator</nav-item>
</c-corporate-header>

Main Content

The sole purpose of the <c-main-content> element is to calculate the distance between the Corporate Header and the Corporate Footer. Using <c-main-content> on your page will ensure you that the footer never slides up the page. If the acutal content on a page is not tall enough to push the footer down and out of the browser viewport, it will be "hanging" on the page with whitespace underneath. 


<c-main-content> 
  <c-main-navigation></c-main-navigation>
  <section>
    <div class="container">

    // Your application goes here. If you are using .container-fluid,

    // your page will take the whole browser width. 

    // Using .container, it will give you margins on wider screens.

    </div>
  </section>
</c-main-content>

Sections and Containers

While c-main-content is calculating the distance between header and footer, you'll need additional elements to set margins and padding. The <section> element is setting both the background colour as well as top and bottom margins. You can override the standard background colour by using the classes .light or .dark. To set boxed left and right margins, use the .container class. For a fluid design use .container-fluid class. A page can be built using several sections with a variety of containers.

Main Navigation

The main navigation has a desktop mode and a mobile mode where the break point for is at 992px viewport width. If you make your browser window narrower or wider you'll see how the navigation changes its appearance. The main navigation is highly customizable as it can host both primary and secondary items as well as nav-items with properties and attributes.

Populate Main navigation via JSON

The Main navigation also supports population of nav items using a JSON object. That's a good alternative when you need on the fly translation of not only your site but also your navigation. Check out this sample page and you'll get a hang of how it's done. 


<c-main-navigation>
  <primary-items>
    <nav-item location="#">Home</nav-item>
    <nav-item location="#" class="active">Style Guide
      <sub-navigation caption="Style Guide">
        <nav-item location="#">Logotype</nav-item>
        <nav-item location="#">Icons</nav-item>
      </sub-navigation>
    </nav-item>  
  </primary-items>
  <secondary-items>
     Secondary items are optional, more information below.
  </secondary-items>
</c-main-navigation>

Primary items

The primary items element is the host for your sites main navigation items. It can contain one or several levels of navigation items.

Nav items

The nav items is where you specify what links you are going to have on your navigation and where they should link to. If you need to access the actual anchor <a> element, it is possible - but not necessary - to add them in the actual nav-item. 


 <nav-item><a href="https://scania.com">Scania.Com</a></nav-item>

Active nav-item

Be sure to set the class="active" on the nav-items that is going to be marked as active. If you have a sub navigation, both the parent nav-item element as well as the sub navigation item should have the active class on them.

Dropdown in a nav-item

If you need to add a dropdown in a nav-item you can use the dropdown="true" attribute to prevent that the page is reloaded.

Attributes and Properties

The nav items supports both attributes and properties.


 <nav-item><a icon="facebook" location="https://www.facebook.com/scaniagroup/" attrs="{ target: '_blank'}" props="{ onclick: function() { window.open('https://www.facebook.com/scaniagroup/' )}}"
 >Scania.Com</a></nav-item>

The "More" transformation

If there isn't engough horizontal space to fit all your primary navigation items, a drop down menu will automaticly be created and some items will be moved there. Please check out our demo page and see how it workes out.

Open Demo (new window)

Sub navigation

Each of the primary navigation items can host their own sub navigation. Be sure that the sub navigation is wrapped within the nav item that is their parent.

<nav-item location="#"class="active">Style Guide <sub-navigation caption="Style Guide"> <nav-item location="#">Logotype</nav-item> <nav-item location="#">Icons</nav-item> </sub-navigation> </nav-item>

Secondary items (optional)

In the <secondary-items> element you can place any custom markup that your solution demands. A common example is the user profile drop down exampled below. In desktop mode, on this page's main navigation, it's the blue box in the upper right hand side of the browser window. If you're in mobile mode, the user profile menu is avaiable among the navigation items on the mobile menu. 


<secondary-items>
  <nav-item dropdown="true"class="active">
    <div class="dropdown"> 
      <div role="button">                 
        <div class="user-profile user-menu dropdown-toggle" data-toggle="dropdown">
          <div class="user-name hidden-md">   
            <span>Jesper Karlsson<span>
            <small>Company name<small>
          </div>
          <div class="user-profile-picture" style="background-image: url('https://scania.github.io/corporate-ui-docs/images/sample-user-image.jpg')"> 
        </div>  
      </div>
      <ul class="dropdown-menu"aria-labelledby="myDropdown">
        <li role="presentation"><a role="menuitem" tabindex="-1"> href="https://idm.scania.com">Change Password</a></li>
        <li role="presentation"><a role="menuitem" tabindex="-1"> href="#">Logout</a></li>
      </ul>        
    </div>
  </nav-item>  
</secondary-items>

Mobile navigation

The desktop version will automaticly re-arrange and be displayed as a traditional mobile navigation with a hamburder icon. Any code included within the secondary items will be included in the mobile navigation but it might need some extra CSS to be displayed correctly below a 992px viewport width. To limit the scope not only to mobile mode, but to the the mobile navigation only, please prefix your selector with .navigation-open


@media (max-width: 991px) { 
.navigation-open .your-selector {
  .your-custom:declaration();
} 
}

Corporate Footer

Basic Example

By default, the <c-corporate-footer> element renders with a copyright text and a wordmark. All you have to do is to include the element as shown in the example.


<c-corporate-footer></c-corporate-footer>

Corporate Footer with Links and Social Icons

By including the social-icons component in the footer you'll be able to promote your local social channels. Add nav-items if you're in need of footer links.

Opening a link in a new browser window or tab

If you prefere to open a nav-item in a new window, please use the JS event onclick="window.open('https://wherever.com')" to do the trick. It's included in the above code sample.

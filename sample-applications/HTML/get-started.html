Introduction

Get started with Corporate UI 4.0.0 Alpha 1. This is a early release that's focused mainly on a solid architecture and making things work in Google Chrome. Please give any feedback using GitHub issues: https://github.com/scania/corporate-ui-dev/issues/new/choose

Quick start

Looking to quickly add Corporate UI to your project? Use our CloudFront CDN. Want to use a package manager? Contact us to participate in the final tests of the NPM package.

Corporate UI CDN Service

Copy-paste the Corporate UI initializer <script> just before the </head> tag.

<script src="https://static.scania.com/build/global/4.0.0-alpha/corporate-ui.js"></script>

About our JavaScripts & CSS

By default, Corporate UI no longer includes any libraries or frameworks in the global scope. Therefore, you will need to add your own libraries and frameworks separatly. In other words, there is by default no Scania styling avilable if you do not say so.

Styling of Custom Elements

All styling of custom elements are done in the build phase and you'll define what theme to apply using the c-theme component. These styles are scoped within each component and will not leak out to other parts of your document.

  <c-theme name="scania"></c-theme>

Starter template

Be sure to have your pages set up with the latest design and development standards. That means using an HTML5 doctype and including a viewport meta tag for proper responsive behaviors. Put it all together and your basic Corporate UI code should look like this:

<!DOCTYPE html>
<html lang="en-GB">
<head>
  <meta http-equiv="X-UA-Compatible" content="IE=edge">  
  <title>Basic Corporate UI 4.0.0 Alpha Template</title>
  <meta charset="utf-8">  
  <meta name="viewport" content="width=device-width, initial-scale=1.0">

  <script src="https://static.scania.com/build/global/4.0.0-alpha/corporate-ui.js"></script>

</head>

<body>

  <c-theme name="scania"></c-theme>

  <c-header
    site-name="Corporate UI 4.0.0 Alpha"
    top-items='[{"text":"Ver 3.5.2", "location": "https://scania.github.io/corporate-ui-docs/developer/bootstrap-original-docs.html"}]'
    primary-items='[{"text":"Get Started", "location": "/" }]'>
  </c-header>

  <c-content>
    <section>
      <h1>Hello World</h1>
    </section>     
  </c-content>

  <c-footer>
    <a href="/cookies" slot="items">Cookies</a>
    <a href="/contact-us" slot="items">Contact us</a>
  </c-footer>

</body>
</html>

Adding global styling

In order to apply the styling availble from previous version of Corporate UI include the stylesheets below.

<link rel="stylesheet" href="https://static.scania.com/vendors/frameworks/bootstrap/3.2.0/dist/css/bootstrap-org.css">
<link rel="stylesheet" href="https://static.scania.com/build/global/3.5.2/css/corporate-ui.css">

Adding Font Awesome 5.3.1 Pro

<link rel="stylesheet" href="https://legend.scania.com/resources/icons/font-awesome/5.3.1/css/all.min.css">

Adding Font Awesome 3.2.1

<link rel="stylesheet" href="https://static.scania.com/resources/fonts/font-awesome/font-awesome.css">

Bootstrap Javascript Components

In order to use Modals, Dropdowns and other components created by the Bootstrap team, we're since version 2.2.0 using Bootstrap Native.

<script src="https://static.scania.com/vendors/frameworks/bootstrap.native/2.0.21/dist/bootstrap-native.min.js"></script>

Other important Javascript files

In the head section we handle what happends both when a user has disabled his browsers Javascript encoding as well as re-directing older browsers to an explaination and instructions of how to install a modern browser.

<noscript><meta http-equiv="refresh" content="0;url=https://static.scania.com/vendors/components/pure-js/nonJS-browsers/1.0.0/nonJS-browsers.html"></noscript>

<script src="https://static.scania.com/vendors/components/pure-js/browser-reject/1.0.0/js/browser-reject.js"></script>
